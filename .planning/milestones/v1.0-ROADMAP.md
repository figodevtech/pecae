# Arquivo de Histórico — ROADMAP Milestone 1 (v1.0)

Este documento registra as tarefas e estimativas originais do roteiro cronológico de desenvolvimento que foram 100% concluídas e validadas ao término da **Milestone 1** do PECAÊ.

## Milestone 1 (Sprint 1) — Fundações: Autenticação & Catálogo Automotivo (Foco: M01 + M04)

### M01 — Autenticação e Cadastro (P0)
**Descrição:** Módulo responsável por todo o ciclo de identidade do usuário: cadastro, autenticação, verificação, recuperação de senha e gestão de sessão. É a fundação de toda a plataforma — todos os módulos dependem deste.

- [x] **M01-T01: Setup de banco, schema Prisma e modelo de usuário** (Prioridade: P0)
  *Nota de Dependency:* O seed de PartCategory deve ser estruturado de forma que o model seja criado apenas no M04.
  *Descrição:* Definir o schema Prisma completo para as entidades de identidade (User, RefreshToken, EmailVerificationToken, PasswordResetToken, OtpCode, TermsAcceptance), criar as migrations no Supabase e configurar o cliente Prisma no NestJS.
    - [x] **M01-T01-ST01:** Escrever schema Prisma — entidades de identidade (Estimativa: 4h)
    - [x] **M01-T01-ST02:** Executar migrations e configurar PrismaService no NestJS (Estimativa: 3h)
    - [x] **M01-T01-ST03:** Seed inicial — PartCategory (lista fixa de categorias de peças) (Estimativa: 3h)
- [x] **M01-T02: Implementar cadastro por e-mail/senha e verificação** (Prioridade: P0)
  *Descrição:* Criar o fluxo completo de cadastro por e-mail e senha no NestJS (AuthController, AuthService, UserService) com validação de DTO via class-validator, hash de senha com bcrypt, criação de conta com status PENDING_VERIFICATION e envio de e-mail de verificação via Resend.
    - [x] **M01-T02-ST01:** Criar RegisterDto, AuthController e AuthService (cadastro e-mail) (Estimativa: 5h)
    - [x] **M01-T02-ST02:** Implementar serviço de e-mail de verificação via Resend (Estimativa: 4h)
    - [x] **M01-T02-ST03:** Refatoração Visual & UX (Estimativa: 6h)
- [x] **M01-T03: Implementar autenticação OAuth (Google) e Telefone (OTP)** (Prioridade: P0)
  *Descrição:* Integrar Google Sign-In no app React Native via expo-auth-session. No NestJS, criar endpoint POST /auth/google que valida o idToken do Google, cria ou recupera o usuário (padrão BUYER) e emite JWT + refresh token.
    - [x] **M01-T03-ST01:** Configurar Google OAuth no Expo e NestJS (Estimativa: 5h)
    - [x] **M01-T03-ST02:** Removido: Apple Sign-In (Sem conta Developer) (Estimativa: 0h)
    - [x] **M01-T03-ST03:** Implementar autenticação por telefone com OTP (Estimativa: 5h)
- [x] **M01-T04: Implementar login, logout, refresh token e recuperação de senha** (Prioridade: P0)
  *Descrição:* Implementar POST /auth/login (e-mail/senha), POST /auth/logout (revogação de refresh token), POST /auth/refresh (rotação de token), POST /auth/forgot-password e POST /auth/reset-password. Implementar JwtStrategy e JwtAuthGuard no NestJS. Criar tela de login no Expo.
    - [x] **M01-T04-ST01:** Implementar POST /auth/login e JwtStrategy no NestJS (Estimativa: 4h)
    - [x] **M01-T04-ST02:** Implementar refresh token rotation e logout (Estimativa: 3h)
    - [x] **M01-T04-ST03:** Implementar recuperação e redefinição de senha (Estimativa: 4h)

### M04 — Catálogo Automotivo (P0)
**Descrição:** Módulo responsável pelo catálogo de veículos da plataforma: marcas, modelos, versões, anos e categorias fixas de peças (PartCategory). É infraestrutura base para os módulos M05 (Cadastro de Sucata) e M07 (Busca e Descoberta). Dados gerenciados exclusivamente pelo administrador.
 
- [x] **M04-T01: Schema Prisma — entidades do catálogo automotivo** (Prioridade: P0)
  *Descrição:* Definir e criar no Prisma as entidades VehicleBrand, VehicleModel, VehicleVersion, VehicleYear e PartCategory com seus relacionamentos hierárquicos, índices de performance e constraints.
    - [x] **M04-T01-ST01:** Escrever schema Prisma — hierarquia Brand → Model → Version → Year (Estimativa: 3h)
    - [x] **M04-T01-ST02:** Escrever model PartCategory e migration do catálogo (Estimativa: 2h)
    - [x] **M04-T01-ST03:** Seed completo de marcas e modelos populares no Brasil (Estimativa: 4h)
- [x] **M04-T02: CatalogController e CatalogService com cache Redis** (Prioridade: P0)
  *Descrição:* Implementar CatalogModule no NestJS com todos os endpoints de leitura do catálogo. Integrar Redis via @nestjs/cache-manager (Upstash) para cache de 24h em todas as respostas. Endpoints públicos (@Public()) pois catálogo não requer autenticação.
    - [x] **M04-T02-ST01:** Criar CatalogModule, CatalogController e CatalogService (Estimativa: 4h)
    - [x] **M04-T02-ST02:** Integrar Redis Upstash como cache provider no NestJS (Estimativa: 3h)
    - [x] **M04-T02-ST03:** Endpoint de autocomplete e busca de modelo para o app (Estimativa: 3h)
- [x] **M04-T03: Seletor em cascata no app React Native** (Prioridade: P0)
  *Descrição:* Implementar o componente de seleção em cascata (Marca → Modelo → Versão → Ano) reutilizável no React Native usando Expo + TanStack Query. Será usado tanto na tela de busca (M07) quanto no wizard de cadastro de sucata (M05).
    - [x] **M04-T03-ST01:** Criar hooks useCatalog para queries com TanStack Query (Estimativa: 3h)
    - [x] **M04-T03-ST02:** Implementar componente VehicleSelector reutilizável (Estimativa: 5h)
    - [x] **M04-T03-ST03:** Admin CRUD do catálogo via endpoints protegidos (Estimativa: 4h)
- [x] **M04-T04: Testes e documentação da API do catálogo** (Prioridade: P1)
  *Descrição:* Criar testes unitários para CatalogService (mock do Prisma) e testes de integração para os endpoints do CatalogController usando supertest. Documentar a API com Swagger (@nestjs/swagger). Testar comportamento do cache.
    - [x] **M04-T04-ST01:** Testes unitários do CatalogService (Estimativa: 3h)
    - [x] **M04-T04-ST02:** Documentação Swagger dos endpoints do catálogo (Estimativa: 2h)
    - [x] **M04-T04-ST03:** Testes de integração dos endpoints de catálogo (Estimativa: 3h)
