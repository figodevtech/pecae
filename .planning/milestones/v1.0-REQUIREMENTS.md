# Arquivo de Histórico — REQUIREMENTS Milestone 1 (v1.0)

Este documento registra os Requisitos Funcionais, Regras de Negócio e Casos de Uso que foram 100% satisfeitos e validados ao término da **Milestone 1** do PECAÊ.

---

## Requisitos Funcionais Satisfeitos

### M01 - Autenticação e Cadastro
- [x] **RF01:** O sistema deve permitir cadastro de usuário com tipo: comprador, vendedor ou ambos (P0)
- [x] **RF02:** O sistema deve permitir autenticação por e-mail e senha (P0)
- [x] **RF03:** O sistema deve permitir autenticação via Google (OAuth 2.0) (P0)
- [x] **RF05:** O sistema deve permitir autenticação via telefone com OTP por SMS (P1)
- [x] **RF06:** O sistema deve exigir verificação de e-mail antes de permitir publicação de anúncios (P0)
- [x] **RF07:** O sistema deve registrar aceite de Termos, Política de Privacidade e LGPD no cadastro (P0)
- [x] **RF08:** O sistema deve permitir recuperação de senha via link seguro por e-mail (P0)

### M04 - Catálogo Automotivo
- [x] **RF20:** Sistema deve listar marcas de veículos ordenadas alfabeticamente (P0)
- [x] **RF21:** Sistema deve listar modelos filtrados por marca selecionada (P0)
- [x] **RF22:** Sistema deve listar versões filtradas por modelo selecionado (P0)
- [x] **RF23:** Sistema deve listar anos disponíveis filtrados por versão selecionada (P0)
- [x] **RF24:** Sistema deve expor lista fixa de PartCategory para seleção no cadastro de sucata (P0)
- [x] **RF25:** Administrador deve poder adicionar, editar e remover marcas, modelos e versões via painel web (P1)
- [x] **RF26:** Sistema deve suportar autocomplete de marca e modelo com debounce de 300ms no app (P1)

## Regras de Negócio Implementadas

### M01 - Autenticação e Cadastro
- [x] **RN-M01-01:** Um e-mail só pode estar associado a uma conta ativa no sistema.
- [x] **RN-M01-02:** Token de verificação de e-mail expira em 24 horas; token de redefinição expira em 1 hora.
- [x] **RN-M01-03:** Todo usuário recém-cadastrado inicia com status PENDING_VERIFICATION.
- [x] **RN-M01-04:** Access token expira em 15 minutos; refresh token em 7 dias com rotation obrigatória.
- [x] **RN-M01-05:** Apenas usuários com status ACTIVE podem publicar anúncios ou iniciar chats.

### M04 - Catálogo Automotivo
- [x] **RN03:** PartCategory é lista fixa gerenciada apenas por administradores — vendedores apenas selecionam.
- [x] **RN-M04-01:** Dados do catálogo são cacheados por 24h no Redis; invalidação manual pelo admin ao editar.
- [x] **RN-M04-02:** Seed inicial obrigatório com marcas mais comuns no Brasil: Fiat, VW, GM, Ford, Toyota, Honda, Hyundai, Renault, Chevrolet, Nissan, Jeep.
